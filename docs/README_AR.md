🎯 **هدف المشروع**
•	**ما هي المشكلة التي يحلها التطبيق:** يوضح كيفية تنفيذ عناصر واجهة مستخدم مخصصة (Custom UI): Toast، Dialog، ActionBar، Toolbar، وتمثيلات رسومية (Custom View).

•	**لماذا هذه المهمة مهمة:** لا تتوافق مكونات أندرويد القياسية دائمًا مع متطلبات التصميم أو الوظائف المحددة. تتيح القدرة على إنشاء مكونات مخصصة واجهة مستخدم فريدة وتفاعلية.

•	**الجمهور المستهدف:** مطورو أندرويد الراغبون في تعلم إنشاء Custom Views وتخصيص واجهة المستخدم بعمق.
________________________________________
📌 **متطلبات التطبيق**

**المتطلبات الوظيفية**
•	عرض إشعارات مخصصة (Toast) مع صور.
•	تشغيل نوافذ حوار (Dialog) معقدة بتخطيطات مخصصة.
•	رسم بياني ديناميكي لمنحنى الجيب (Sinus) في الوقت الفعلي.
•	توضيح العمل مع Toolbar و ActionBar مخصصين.

**المتطلبات غير الوظيفية**
•	**الأداء:** رسم رسومي سلس (60 إطارًا في الثانية لمنحنى الجيب).
•	**سهولة الاستخدام:** واجهة شاشة رئيسية بسيطة للانتقال إلى أمثلة العرض.
•	**الموثوقية:** التعامل الصحيح مع دورة حياة الـ Activity أثناء إعادة الرسم الرسومي.
________________________________________
🧠 **الهندسة المعمارية العامة**
•	**النهج المختار:**  
مكونات أندرويد القياسية (نظام يعتمد على Activity و View).

•	**لماذا تم اختياره:**  
بالنسبة لمشروع يركز على مكتبة واجهة مستخدم/أمثلة، فإن الوصول المباشر إلى View و Activity هو الطريقة الأكثر فعالية لتوضيح آليات الرسم والتخصيص.

•	**مكونات النظام الرئيسية:**  
MainActivity، Custom Views (CustomSinusView)، فئات المساعدة الثابتة (MyToast).

🧩 **وصف تفصيلي للفئات**

📌 **الفئة: MainActivity**
**الدور:**  
الشاشة الرئيسية للتطبيق (القائمة).

**المسؤولية:**  
تهيئة واجهة المستخدم والتنقل عبر أقسام التطبيق.

**الطرق الرئيسية:** 
- `onCreate()` — تهيئة الأزرار وتعيين مستمعي الأحداث.
- `initComponents()` — البحث عن عناصر الـ View بواسطة المعرف (ID).

**التفاعل مع الفئات الأخرى:**  
تستدعي الطرق الثابتة لـ `MyToast` وتطلق `DinamicSinus` و `AddCustomActionBar` و `MyToolbar`.
________________________________________
📌 **الفئة: CustomSinusView**
**الدور:**  
مكون رسومي مخصص.

**لماذا يتم استخدامه:**  
لتنفيذ الرسم الديناميكي للدوال الرياضية، وهو أمر غير ممكن باستخدام أدوات أندرويد القياسية.

**الطرق الرئيسية:**
- `onDraw()` — منطق رسم منحنى الجيب على الـ Canvas.
- `postInvalidateDelayed()` — إنشاء حلقة رسوم متحركة.
________________________________________
📌 **الفئة: MyToast**
**الدور:**  
فئة مساعدة لواجهة المستخدم.

**المسؤولية:**  
عرض إشعارات منبثقة بتخطيط مخصص (نص + أيقونة).
________________________________________
🔄 **مخطط سير عمل التطبيق**

**سيناريو (عرض الرسم البياني):**
1. ينقر المستخدم على "Sinus" في MainActivity.
2. يطلق النظام الـ Activity الخاص بـ DinamicSinus.
3. يقوم DinamicSinus بتحميل التخطيط الذي يحتوي على CustomSinusView.
4. يبدأ CustomSinusView دورة الرسم عبر `onDraw` مع زيادة عدد النقاط المعروضة تدريجيًا.

🎨 **تحليل UI/UX**
•	**لماذا صممت الواجهة بهذا الشكل:**  
الواجهة مبسطة قدر الإمكان (قائمة أزرار) لعدم تشتيت الانتباه عن جوهر كل مثال محدد.

•	**المبادئ المستخدمة:**
–	**البساطة:** الوصول إلى جميع الوظائف بنقرة واحدة.
–	**المنطقية:** تم نقل كل مثال إلى شاشة أو حوار منفصل.
–	**سهولة الوصول:** استخدام ألوان متباينة وأيقونات واضحة.

•	**ما الذي يمكن تحسينه:**  
إضافة دعم المظهر الداكن (Dark Theme) وتحسين العرض للأجهزة اللوحية.
________________________________________
⚙️ **العمل مع الخيوط (Threading)**
**الوصف:** 
- المستخدم: `postInvalidateDelayed` (آلية داخلية للـ View للتفاعل مع خيط واجهة المستخدم).
- **لماذا تم اختيار هذه الطريقة:**  
تضمن مزامنة الرسم مع معدل تحديث الشاشة دون الحاجة إلى إدارة يدوية للخيوط.

•	**كيفية منع:**
–	**التجمد (ANR):** تم تحسين كود الرسم، والحسابات في حدها الأدنى.
–	**تسرب الذاكرة:** استخدام سياق التطبيق (Application Context) في Toast وإنهاء الرسوم المتحركة بشكل صحيح.
