🎯 **מטרת הפרויקט**
•	**איזו בעיה האפליקציה פותרת:** הדגמת מימוש של אלמנטים מותאמים אישית בממשק המשתמש (Custom UI): Toast, Dialog, ActionBar, Toolbar ותצוגות גרפיות (Custom View).

•	**למה המשימה הזו חשובה:** רכיבי אנדרואיד סטנדרטיים לא תמיד מתאימים לעיצוב או לדרישות הפונקציונליות של האפליקציה. היכולת ליצור רכיבים מותאמים אישית מאפשרת יצירת ממשק משתמש ייחודי ואינטראקטיבי.

•	**קהל יעד:** מפתחי אנדרואיד המעוניינים ללמוד על Custom Views והתאמה עמוקה של ה-UI.
________________________________________
📌 **דרישות האפליקציה**

**דרישות פונקציונליות**
•	הצגת הודעות קופצות (Toast) מותאמות אישית עם תמונות.
•	הפעלת חלונות דיאלוג מורכבים עם עיצוב מוגדר משתמש.
•	ציור גרף סינוס דינמי בזמן אמת.
•	הדגמת מימוש של Toolbar ו-ActionBar מותאמים אישית.

**דרישות לא פונקציונליות**
•	**ביצועים:** רינדור גרפי חלק (60 FPS עבור גרף הסינוס).
•	**חווית משתמש:** ממשק מסך ראשי פשוט למעבר בין דוגמאות ההדגמה.
•	**אמינות:** טיפול נכון במחזור החיים של ה-Activity בזמן ציור מחדש של הגרפיקה.
________________________________________
🧠 **ארכיטקטורה כללית**
•	**הגישה שנבחרה:**  
Standard Android Components (Activity-driven / View-oriented).

•	**למה היא נבחרה:**  
עבור פרויקט הממוקד בספריית UI/דוגמאות, גישה ישירה ל-View ול-Activity היא הדרך היעילה ביותר להדגים מנגנוני רינדור והתאמה אישית.

•	**מרכיבי המערכת העיקריים:**  
MainActivity, Custom Views (CustomSinusView), Static Helper Classes (MyToast).

🧩 **תיאור מחלקות מפורט**

📌 **מחלקה: MainActivity**
**תפקיד:**  
המסך הראשי של האפליקציה (תפריט).

**אחריות:**  
אתחול ה-UI וניווט בין חלקי האפליקציה.

**שיטות עיקריות:** 
- `onCreate()` — מאתחלת כפתורים ומגדירה מאזינים לאירועים.
- `initComponents()` — מוצאת אלמנטים של View לפי ID.

**אינטראקציה עם מחלקות אחרות:**  
קוראת לשיטות סטטיות של `MyToast`, מפעילה את `DinamicSinus`, `AddCustomActionBar`, `MyToolbar`.
________________________________________
📌 **מחלקה: CustomSinusView**
**תפקיד:**  
רכיב גרפי מותאם אישית.

**למה היא בשימוש:**  
למימוש רינדור דינמי של פונקציות מתמטיות, דבר שלא אפשרי עם כלי אנדרואיד סטנדרטיים.

**שיטות עיקריות:**
- `onDraw()` — לוגיקה לציור גרף הסינוס על ה-Canvas.
- `postInvalidateDelayed()` — יצירת לולאת אנימציה.
________________________________________
📌 **מחלקה: MyToast**
**תפקיד:**  
מחלקת עזר ל-UI.

**אחריות:**  
הצגת הודעות קופצות עם Layout מותאם אישית (טקסט + אייקון).
________________________________________
🔄 **דיאגרמת זרימת עבודה**

**תרחיש (צפייה בגרף):**
1. המשתמש לוחץ על "Sinus" ב-MainActivity.
2. המערכת מפעילה את Activity ה-DinamicSinus.
3. DinamicSinus טוען את ה-Layout המכיל את CustomSinusView.
4. CustomSinusView מתחיל את מחזור הציור דרך `onDraw`, ומגדיל בהדרגה את מספר הנקודות המוצגות.

🎨 **ניתוח UI/UX**
•	**למה הממשק עוצב כך:**  
הממשק פשוט ככל האפשר (תפריט כפתורים) כדי לא להסיח את הדעת מהמהות של כל דוגמה ספציפית.

•	**עקרונות בשימוש:**
–	**פשטות:** גישה לכל הפונקציות בלחיצה אחת.
–	**לוגיקה:** כל דוגמה מועברת למסך או דיאלוג נפרד.
–	**נגישות:** שימוש בצבעים בעלי ניגודיות גבוהה ואייקונים ברורים.

•	**מה ניתן לשפר:**  
הוספת תמיכה בערכת נושא כהה (Dark Theme) ואופטימיזציה לטאבלטים.
________________________________________
⚙️ **עבודה עם תהליכונים (Threading)**
**תיאור:** 
- שימוש ב: `postInvalidateDelayed` (מנגנון פנימי של View לאינטראקציה עם UI thread).
- **למה נבחרה שיטה זו:**  
מבטיחה סנכרון ציור עם קצב רענון המסך ללא צורך בניהול ידני של תהליכונים.

•	**מניעת:**
–	**תקיעות (ANR):** קוד הציור מותאם, החישובים מינימליים.
–	**דליפות זיכרון:** שימוש ב-Context של האפליקציה ב-Toast וסיום נכון של האנימציות.
