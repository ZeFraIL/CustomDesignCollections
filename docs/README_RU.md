🎯 **Цель проекта**
•	**Какую задачу решает приложение:** Демонстрация реализации кастомных (пользовательских) элементов интерфейса: Toast, Dialog, ActionBar, Toolbar и графических представлений (Custom View).

•	**Почему эта задача важна:** Стандартные компоненты Android не всегда соответствуют дизайну или функциональным требованиям приложения. Умение создавать собственные компоненты позволяет делать интерфейс уникальным и интерактивным.

•	**Целевая аудитория:** Разработчики Android, желающие изучить создание Custom Views и глубокую кастомизацию UI.
________________________________________
📌 **Требования к приложению**

**Функциональные требования**
•	Отображение кастомных уведомлений (Toast) с изображениями.
•	Запуск сложных диалоговых окон с пользовательской разметкой.
•	Отрисовка динамического графика синусоиды в реальном времени.
•	Демонстрация работы с кастомными Toolbar и ActionBar.

**Нефункциональные требования**
•	**Производительность:** Плавная отрисовка графики (60 FPS для синусоиды).
•	**Удобство использования:** Простой интерфейс главного экрана для перехода к демо-примерам.
•	**Надёжность:** Корректная обработка жизненного цикла Activity при перерисовке графики.
________________________________________
🧠 **Общая архитектура**
•	**Выбранный подход:**  
Standard Android Components (Activity-driven / View-oriented).

•	**Почему выбран именно он:**  
Для проекта, ориентированного на UI-библиотеку/примеры, прямой доступ к View и Activity является наиболее эффективным для демонстрации механизмов отрисовки и кастомизации.

•	**Основные компоненты системы:**  
MainActivity, Custom Views (CustomSinusView), Static Helper Classes (MyToast).

🧩 **Подробное описание классов**

📌 **Класс: MainActivity**
**Роль:**  
Главный экран приложения (Меню).

**Ответственность:**  
Инициализация UI и навигация по разделам приложения.

**Основные методы:** 
- `onCreate()` — инициализация кнопок и установка слушателей событий.
- `initComponents()` — поиск View-элементов по ID.

**Взаимодействие с другими классами:**  
Вызывает статические методы `MyToast`, запускает `DinamicSinus`, `AddCustomActionBar`, `MyToolbar`.
________________________________________
📌 **Класс: CustomSinusView**
**Роль:**  
Пользовательский графический компонент.

**Почему используется:**  
Для реализации динамической отрисовки математических функций, которая невозможна стандартными средствами Android.

**Основные методы:**
- `onDraw()` — логика рисования синусоиды на Canvas.
- `postInvalidateDelayed()` — создание цикла анимации.
________________________________________
📌 **Класс: MyToast**
**Роль:**  
Утилитарный класс для UI.

**Ответственность:**  
Отображение всплывающих уведомлений с кастомным Layout (текст + иконка).
________________________________________
🔄 **Диаграмма работы приложения**

**Сценарий (Просмотр графика):**
1. Пользователь нажимает "Sinus" в MainActivity.
2. Система запускает DinamicSinus Activity.
3. DinamicSinus загружает макет с CustomSinusView.
4. CustomSinusView начинает цикл отрисовки через `onDraw`, постепенно увеличивая количество отображаемых точек.

🎨 **UI/UX анализ**
•	**Почему интерфейс сделан именно так:**  
Интерфейс максимально упрощен (кнопочное меню), чтобы не отвлекать от сути каждого конкретного примера.

•	**Какие принципы использованы:**
–	**Простота:** Доступ ко всем функциям в один клик.
–	**Логичность:** Каждый пример вынесен в отдельный экран или диалог.
–	**Доступность:** Использование контрастных цветов и понятных иконок.

•	**Что можно улучшить:**  
Добавить темную тему и поддержку планшетов.

⚙️ **Работа с потоками**
**Опишите:** 
- Используются: `postInvalidateDelayed` (внутренний механизм View для работы с UI-потоком).
- **Почему выбран этот способ:**  
Обеспечивает синхронизацию отрисовки с частотой обновления экрана без необходимости ручного управления потоками.

•	**Как предотвращаются:**
–	**зависания (ANR):** Код отрисовки оптимизирован, вычисления минимальны.
–	**утечки памяти:** Использование контекста приложения в Toast и корректное завершение анимаций.
