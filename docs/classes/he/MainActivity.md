# תיאור מחלקת MainActivity

________________________________________
## 1. מידע כללי
*   **שם המחלקה:** `MainActivity`
*   **סוג:** Activity (פעילות)
*   **מטרת המחלקה:** המסך הראשי של האפליקציה. משמש כנקודת כניסה ותפריט ניווט להדגמת אלמנטים שונים של ממשק משתמש מותאם אישית.
*   **אינטראקציה:** מפעילה Activities אחרים (`DinamicSinus`, `AddCustomActionBar`, `MyToolbar`) וקוראת למחלקות עזר (`MyToast`, `AlertDialog`).

________________________________________
## 2. משתנים (שדות המחלקה)
| שם | סוג | מטרה | היכן בשימוש |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | שומר התייחסות לסביבת המערכת. | מועבר ל-Toast, Dialog ו-Intent. |
| `bToast` | `Button` | כפתור להצגת Toast מותאם אישית. | `initComponents()`, `onCreate()` |
| `bDialog` | `Button` | כפתור לפתיחת דיאלוג מורכב. | `initComponents()`, `onCreate()` |
| `bSinus` | `Button` | כפתור למעבר לגרף הסינוס. | `initComponents()`, `onCreate()` |
| `bActionBar` | `Button` | כפתור למעבר ל-ActionBar מותאם אישית. | `initComponents()`, `onCreate()` |
| `bToolbar` | `Button` | כפתור למעבר ל-Toolbar מותאם אישית. | `initComponents()`, `onCreate()` |

________________________________________
## 3. מתודות המחלקה

### מתודה: onCreate
*   **סוג:** public
*   **ערך חזרה:** void
*   **פרמטרים:**
    | שם | סוג | תיאור |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | מצב מערכת שמור (אם קיים). |
*   **מה המתודה עושה:**  
    1. קוראת למתודת האב `super.onCreate`.
    2. קובעת את פריסת הממשק `R.layout.activity_main`.
    3. קוראת ל-`initComponents()` לקישור הכפתורים.
    4. מקצה מאזיני לחיצה (`setOnClickListener`) לכל הכפתורים. בלחיצה על `bDialog`, נוצר ומוצג `AlertDialog` מורכב עם שדות קלט וכפתורי פעולה (SMS, אימייל וכו').
*   **מתי נקראת:** כאשר המערכת יוצרת את הפעילות.
*   **מה חשוב להבין:** כל לוגיקת האתחול מרוכזת כאן. יצירת דיאלוג בתוך מתודה זו הופכת את הקוד למסורבל; עדיף היה להעביר את יצירת הדיאלוג למתודה נפרדת.

### מתודה: initComponents
*   **סוג:** private
*   **ערך חזרה:** void
*   **מה המתודה עושה:**  
    מקשרת בין משתני ה-Java לאלמנטים בפריסת ה-XML באמצעות `findViewById`. מאתחלת גם את ה-`context` עם `this`.
*   **מתי נקראת:** בתוך `onCreate`.

________________________________________
## 4. מחזור חיים
*   **onCreate():** נקראת ראשונה. כאן קורה "הולדת" המסך: הגדרת העיצוב וקישור הכפתורים.

________________________________________
## 8. הסבר פשוט
**הסבר במילים פשוטות:**  
דמיינו ש-`MainActivity` היא **התפריט הראשי במשחק**. יש לכם כפתורים כמו "התחל משחק", "הגדרות", "יציאה". כל כפתור מוביל לחלק אחר. `MainActivity` עצמה לא משחקת את המשחק; היא רק עוזרת לכם לבחור את המצב הרצוי.
