# Описание класса MainActivity

________________________________________
## 1. Общая информация
*   **Имя класса:** `MainActivity`
*   **Тип:** Activity (Активность)
*   **Назначение:** Главный экран приложения. Служит точкой входа и навигационным меню для демонстрации различных кастомных элементов интерфейса.
*   **Взаимодействие:** Запускает другие Activity (`DinamicSinus`, `AddCustomActionBar`, `MyToolbar`) и вызывает вспомогательные классы (`MyToast`, `AlertDialog`).

________________________________________
## 2. Переменные (поля класса)
| Имя | Тип | Назначение | Где используется |
| :--- | :--- | :--- | :--- |
| `context` | `Context` | Хранит ссылку на окружение системы. | Передается в Toast, Dialog и Intent. |
| `bToast` | `Button` | Кнопка для показа кастомного Toast. | `initComponents()`, `onCreate()` |
| `bDialog` | `Button` | Кнопка для открытия сложного диалога. | `initComponents()`, `onCreate()` |
| `bSinus` | `Button` | Кнопка перехода к графику синуса. | `initComponents()`, `onCreate()` |
| `bActionBar` | `Button` | Кнопка перехода к кастомному ActionBar. | `initComponents()`, `onCreate()` |
| `bToolbar` | `Button` | Кнопка перехода к кастомному Toolbar. | `initComponents()`, `onCreate()` |

________________________________________
## 3. Методы класса

### Метод: onCreate
*   **Тип метода:** public
*   **Возвращаемое значение:** void (ничего)
*   **Параметры:**
    | Имя | Тип | Описание |
    | :--- | :--- | :--- |
    | `savedInstanceState` | `Bundle` | Сохраненное состояние системы (если есть). |
*   **Что делает метод:**  
    1. Вызывает родительский метод `super.onCreate`.
    2. Устанавливает макет интерфейса `R.layout.activity_main`.
    3. Вызывает `initComponents()` для привязки кнопок.
    4. Назначает слушатели нажатий (`setOnClickListener`) для всех кнопок. При нажатии на `bDialog` создается и отображается сложный `AlertDialog` с полями ввода и кнопками действий (SMS, Email и др.).
*   **Когда вызывается:** При создании активности системой.
*   **Что важно понимать:** Здесь сосредоточена вся логика инициализации. Создание диалога внутри этого метода делает код громоздким; лучше было бы вынести создание диалога в отдельный метод.

### Метод: initComponents
*   **Тип метода:** private
*   **Возвращаемое значение:** void
*   **Параметры:** отсутствуют.
*   **Что делает метод:**  
    Связывает переменные Java с элементами XML-разметки с помощью `findViewById`. Также инициализирует `context` значением `this`.
*   **Когда вызывается:** Внутри `onCreate`.
*   **Что важно понимать:** Метод помогает держать код `onCreate` более чистым.

________________________________________
## 4. Жизненный цикл
*   **onCreate():** Вызывается первым. Здесь происходит "рождение" экрана: установка дизайна и привязка кнопок.

________________________________________
## 5. Взаимодействие с интерфейсом (UI)
*   **Элементы:** Кнопки (`Button`), Текстовые поля (`TextView`, `EditText` — внутри диалога).
*   **Связь:** Через `findViewById`.
*   **События:** Обработка кликов (`setOnClickListener`).

________________________________________
## 6. Взаимодействие с другими компонентами
*   **Переходы:** Использует `Intent` для запуска `DinamicSinus`, `AddCustomActionBar`, `MyToolbar`.
*   **Данные:** В текущем коде данные между экранами не передаются, только факт запуска.

________________________________________
## 7. Общая логика класса
Класс работает как диспетчер. Пользователь видит список кнопок, каждая из которых демонстрирует отдельную функцию приложения. При клике либо выполняется действие прямо на месте (Toast, Dialog), либо открывается новый экран.

**Пример использования:**
1. Пользователь запускает приложение.
2. Нажимает кнопку "Dialog".
3. Появляется окно с просьбой ввести данные.
4. Пользователь вводит номер и нажимает "Cancel" для выхода из окна.

________________________________________
## 8. Упрощенное объяснение
**Объяснение простыми словами:**  
Представьте, что `MainActivity` — это **главное меню в игре**. У вас есть кнопки "Начать игру", "Настройки", "Выход". Каждая кнопка ведет в свой раздел. Сама `MainActivity` не играет в игру, она только помогает вам выбрать нужный режим.

________________________________________
**Совет по коду:**  
Логика создания `AlertDialog` очень большая. Рекомендуется вынести её в отдельный метод (например, `showContactDialog()`), чтобы `onCreate` был коротким и понятным.
